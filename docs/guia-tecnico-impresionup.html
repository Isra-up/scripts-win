Guía de Instalación — Cola IMPRESIONUP
Kyocera TASKalfa MZ2501ci KX Windows 11 IP: 10.1.6.63 Versión 1.0 · 2026-06-23
Antes de empezar
  • USB con la carpeta IMPRESIONUP preparada
  • Acceso de administrador en el equipo
  • El equipo conectado a la red de la UP
  • Windows 11 instalado en el equipo
Estructura de la USB

La USB debe tener esta estructura exacta. No muevas ni renombres ningún archivo.

IMPRESIONUP\
    ├── install-impresionup.ps1  ← el script
    └── Kyocera_64bit\  ← carpeta completa de drivers
        ├── OEMSETUP.INF
        └── (resto de archivos del driver)
Pasos de instalación
01
Inserta la USB en el equipo
Anota la letra de unidad que Windows le asigna. La verás en el Explorador de archivos — puede ser D:, E: u otra letra.
02
Abre PowerShell como Administrador
Presiona Inicio, escribe PowerShell, haz clic derecho sobre el resultado y selecciona "Ejecutar como administrador". Confirma en el aviso de UAC.
Importante
Debe decir "Administrador" en la barra de título de la ventana. Si no lo dice, ciérrala y repite este paso.
03
Navega a la carpeta del script en la USB
Reemplaza D: por la letra que viste en el paso 01.
PS> cd D:\IMPRESIONUP
04
Ejecuta el script
PS> .\install-impresionup.ps1
Si aparece error de política de ejecución
Ejecuta primero el siguiente comando, responde S cuando pregunte, y luego vuelve a correr el script del paso anterior.
PS> Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
05
Verifica que la impresora quedó instalada
Ve a Configuración → Bluetooth y dispositivos → Impresoras y escáneres. Debe aparecer IMPRESIONUP en la lista.
Prueba de impresión
Haz clic en IMPRESIONUP → Propiedades de la impresora → botón Imprimir página de prueba. Si imprime, la instalación fue exitosa.
Lo que verás en pantalla

Así debe verse la salida del script cuando la instalación es exitosa. Compara con lo que aparece en tu pantalla.

Windows PowerShell — Administrador
>> Verificando archivo de driver...    OK   >> Instalando driver Kyocera en Windows (pnputil)...    AVISO: Driver ya existente o advertencia — continuando.   >> Creando puerto LPR 'IMPRESIONUP' -> 10.1.6.63 ...    OK   >> Verificando instalación previa de 'IMPRESIONUP'...    OK   >> Agregando impresora 'IMPRESIONUP'...    OK   ============================================   IMPRESIONUP instalada correctamente.   IP: 10.1.6.63 | Puerto LPR: IMPRESIONUP ============================================
Normal
El mensaje "Driver ya existente o advertencia — continuando" en amarillo es normal — significa que el driver ya estaba en Windows o que pnputil dio un aviso no crítico. El proceso continúa sin problema.
Solución de problemas
Mensaje de error Causa probable Qué hacer
ERROR: No se encontró: ...\OEMSETUP.INF La carpeta Kyocera_64bit no está junto al script o tiene otro nombre Verifica la estructura de la USB. La carpeta debe llamarse exactamente Kyocera_64bit.
ERROR: Ejecuta el script como Administrador PowerShell no tiene privilegios de administrador Cierra PowerShell y ábrelo de nuevo con clic derecho → Ejecutar como administrador.
El término '.\install...' no se reconoce No estás en la carpeta correcta de la USB Ejecuta cd D:\IMPRESIONUP (ajusta la letra) y vuelve a intentar.
ERROR: en el puerto o Add-Printer El puerto o impresora anterior no se pudo eliminar Ve a Panel de control → Dispositivos e impresoras, elimina manualmente cualquier entrada IMPRESIONUP y vuelve a ejecutar el script.
Verificación final

Marca cada punto antes de pasar al siguiente equipo

IMPRESIONUP aparece en Configuración → Impresoras y escáneres Si no aparece, el script terminó con error — revisa la sección de problemas.
La página de prueba se imprimió correctamente Propiedades de la impresora → Imprimir página de prueba.
El script terminó con el mensaje verde de éxito "IMPRESIONUP instalada correctamente."
