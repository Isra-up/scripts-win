Cómo funciona

La memoria de Claude Code es una carpeta local. En cada equipo configurado, esa carpeta es en realidad un symlink que apunta al repo privado en GitHub. Claude escribe ahí directamente — tú decides cuándo sincronizar.

Claude Code ~/.claude/.../memory
→
Symlink ~/proyectos/
claude-memory
→
GitHub (privado) Isra-up/
claude-memory
→
Otro equipo ~/proyectos/
claude-memory
Regla: Claude escribe solo. Tú sincronizas manualmente al terminar y al iniciar cada sesión de trabajo.
Al terminar En cualquier equipo

Guardar y subir la memoria

Ejecuta esto en la terminal antes de cerrar Claude Code o apagar el equipo.

$ cd ~/proyectos/claude-memory
$ git add .
$ git commit -m "sync memoria"
$ git push

Si no hubo cambios en la memoria durante la sesión, Git te avisará — no pasa nada, no se crea un commit vacío.

Al iniciar En el otro equipo

Traer la memoria actualizada

Antes de abrir Claude Code en el equipo donde vas a trabajar ahora.

$ cd ~/proyectos/claude-memory
$ git pull
Orden importa: siempre sube (git push) en el equipo donde terminaste antes de bajar (git pull) en el equipo donde vas a continuar.
Setup inicial Una sola vez por equipo

Configurar un equipo nuevo

Realiza estos pasos solo la primera vez en cada equipo.

  • Clonar el repositorio de memoria en el equipo
  • Abrir Claude Code una vez para que cree su estructura de carpetas
  • Ver qué carpeta creó Claude — anota el nombre exacto
  • Reemplazar esa carpeta por un symlink al repo
# 1. Clonar el repo
$ git clone git@github.com:Isra-up/claude-memory.git ~/proyectos/claude-memory

# 2. Abrir Claude Code una vez, luego ver qué carpeta creó
$ ls ~/.claude/projects/
# Verás algo como: -home-miusuario-carpeta

# 3. Eliminar la carpeta de memoria y crear el symlink
$ rm -rf ~/.claude/projects/<carpeta>/memory
$ ln -s ~/proyectos/claude-memory ~/.claude/projects/<carpeta>/memory
Verifica: ls ~/.claude/projects/<carpeta>/memory debe mostrar los archivos del repo, no una carpeta vacía.
